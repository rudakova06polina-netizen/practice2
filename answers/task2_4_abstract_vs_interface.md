# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование                                                                                                                         |
|----------|---------|-------------------------------------------------------------------------------------------------------------------------------------|
| Все животные имеют имя и возраст, но звучат по-разному |Абстрактный класс| Есть общее состояние (имя, возраст) + отношение «является»|
| Некоторые объекты (документ, изображение, настройки) можно сериализовать |Интерфейс| Отношение «умеет» — объект умеет сериализоваться. Объекты из разных иерархий не связаны между собой. Нужна множественная реализация |
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному |Абстрактный класс| Есть общее состояние + отношение «является». Абстрактный метод move() задаёт разный способ движения|
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi |Интерфейс| Отношение «умеет» — умеет подключаться к Wi-Fi. Объекты из совершенно разных иерархий, у них нет общего состояния.|
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет |Абстрактный класс| Есть общее поле (цвет) + отношение «является». Абстрактные методы area() и perimeter() требуют разной реализации для каждой фигуры|
| Класс должен одновременно уметь летать, плавать и ездить |Интерфейс |   Нужна множественная реализация, а Java не поддерживает множественное наследование классов|
